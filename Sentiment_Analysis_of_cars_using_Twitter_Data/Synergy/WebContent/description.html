SENTIMENT ANALYSIS OF CARS USING TWITTER DATA

Project Description

  1. Project Purpose

1. The purpose of the project is to analyze the sentiments of the users on different cars using Twitter data.

2. Different car models are queried on the internet and the data related to them is retrieved in the form of Tweets through Twitter Search API.

3. The retrieved tweets are then analyzed and the classification algorithms are run on the data and the sentiment is then pulled out from the data.


  1. Project Flow

1. Collection of Tweets: The authentication and extraction of tweets is done by using Java.
2. Data Cleansing and Transformation: The next step is to cleanse the received tweets. R queries are used to filter and cleanse the data.
3. Mining and Processing using R: The cleansed tweets are then directly processed in R to project the final sentiment score graphically. The histograms, bar charts and word cloud are used to present the sentiment for each car model.
4. Visualization: To present the overall sentiment for various car models, dashboard is used.

A comparative analysis is done for each car model.

The overall system is scalable i.e., as the user input in terms of tweets is increased, the system generates results and behaves accordingly.


  1. Data Tools

1. R Programming Language: This language is used for statistical programming. It has pre compiled binary versions provided for each machine.
2. R Studio: It is a free and open source interface for R. It provides multiple platforms. For our project, we have used R Studio desktop where we run the program locally on our machines.
3. Mongo DB: Mongo db is a noSQL database. It is a document database that provides high performance, high availability and easy scalability. It stores its data in files and uses memory mapped files for data management and efficiency. This project uses mongo DB as the data store to store all the real time tweets and posts. Monog DB can also be configured for data replication which serves a major purpose in maintaining consistency of data.
4. Shiny: Shiny is a web framework and R has package named Shiny. Shiny integrates the statistical and computational features of R well with the modern web. Shiny enables interactive web applications by processing and analyzing the parameters that are user friendly capabilities. It also offers a wide range of outputs such as tables, plots etc.
